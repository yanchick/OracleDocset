1.79 DDL_LOCK_TIMEOUT

Property Description

Parameter type

Integer

Default value

0

Modifiable

ALTER SESSION

Modifiable in a PDB

Yes

Range of values

0 to 1,000,000 (in seconds)

Basic

No

DDL_LOCK_TIMEOUT specifies a time limit for how long DDL statements will wait in a DML lock queue. A value of zero indicates a status of NOWAIT. The maximum value of 1,000,000 seconds will result in the DDL statement waiting forever to acquire a DML lock.

If a lock is not acquired before the timeout period expires, then an error is returned.
