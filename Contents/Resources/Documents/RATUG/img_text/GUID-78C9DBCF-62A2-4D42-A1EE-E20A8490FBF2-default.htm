Description of the illustration dbr_capture_review.gif

On the right of the page, the following text appears: Step 6 of 6. The Back button appear to the left of the text, and the Submit and Cancel buttons appear to the right of the text.

The Database section at the top of the page includes a Concurrent Capture field that has a value of No.

Below the Concurrent Capture field, the Capture Databases table includes the following columns: Database Target, Capture Name, Intermediate Storage Root Location, Database Host, and Description.

The single row entry has the following column values: the Database Target is database, the Name is NewEmpCapture, the Intermediate Storage Root Location is /scratch/destination_dir, the Database Host is hostname.example.com, and the Description is blank.

Below the Options section is the Database section. It includes the SQL Performance Analyzer, Workload Filters, Storage, and Scheduler sections.

The SQL Performance Analyzer section includes the Capture SQL statements into a SQL Tuning Set during workload capture field, which is selected.

The Workload Filters section includes the Excluded Sessions table, which has the Filter Name, Type, Session Attribute, and Value columns. There are two rows in the table, which have these values:

Row 1 has values of Oracle Management Service (DEFAULT), Excluded, Program, and OMS.

Row 2 has values of Oracle Management Service (DEFAULT), Excluded, Module, and emagent%.

The Storage section includes the text "Use intermediate storage location as final storage location."

The Schedule section includes the Capture Schedule and Automatic Workload Repository sections.

In the Capture Schedule section, the Start field has a value of "Immediately" and the Duration field has a value of "For 5 minutes."

In the Automatic Workload Repository section, the Start field has a value of "Immediately."

End of description.
