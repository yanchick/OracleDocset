Sample LogMiner Configuration

Figure 20-1 shows a sample LogMiner configuration. In this figure, the source database in Boston generates redo log files that are archived and shipped to a database in San Francisco. A LogMiner dictionary has been extracted to these redo log files. The mining database, where LogMiner will actually analyze the redo log files, is in San Francisco. The Boston database is running Oracle Database 11g and the San Francisco database is running Oracle Database 12c.

Figure 20-1 Sample LogMiner Database Configuration


Description of "Figure 20-1 Sample LogMiner Database Configuration"

Figure 20-1 shows just one valid LogMiner configuration. Other valid configurations are those that use the same database for both the source and mining database, or use another method for providing the data dictionary. These other data dictionary options are described in "LogMiner Dictionary Options".
